# Memory System

Cortex 的记忆系统基于潜意识架构，提供自动化的知识管理和上下文构建。

## 架构概览

```
┌─────────────────┐    ┌──────────────────┐    ┌─────────────────┐
│   Working       │    │   Subconscious   │    │   Knowledge     │
│   Memory        │◄──►│   Memory         │◄──►│   Graph         │
│                 │    │                  │    │                 │
│ • 当前会话      │    │ • 观察记录       │    │ • 实体关系      │
│ • Token 预算    │    │ • 反思洞察       │    │ • 概念连接      │
│ • 上下文窗口    │    │ • 偏好追踪       │    │ • 语义索引      │
└─────────────────┘    └──────────────────┘    └─────────────────┘
```

## 自动触发机制

### 1. 观察记录 (Observation)

**触发条件**：
- 用户输入新消息
- Agent 完成工具调用
- 文件系统发生变更
- 错误或异常发生

**自动行为**：
- 提取关键信息（实体、动作、结果）
- 记录时间戳和上下文
- 更新知识图谱中的相关节点

### 2. 反思处理 (Reflection)

**触发条件**：
- 会话达到特定长度阈值
- 检测到重复模式或问题
- 用户明确请求总结
- 会话结束或分支创建

**自动行为**：
- 分析会话中的成功/失败模式
- 提取可复用的解决方案
- 更新用户偏好和工作习惯
- 生成洞察并存储到潜意识记忆

### 3. 偏好学习 (Preference Tracking)

**触发条件**：
- 用户选择特定工具或方法
- 用户修正或拒绝建议
- 重复的工作流程模式
- 明确的偏好表达

**自动行为**：
- 记录用户的选择偏好
- 调整未来建议的权重
- 更新个性化配置
- 影响上下文构建策略

### 4. 知识图谱更新 (Knowledge Graph)

**触发条件**：
- 新实体或概念出现
- 实体间关系发生变化
- 代码结构或项目架构更新
- 外部知识引入

**自动行为**：
- 创建或更新实体节点
- 建立或修改关系边
- 计算语义相似度
- 更新索引和检索权重

## 配置与控制

### Token 预算管理

```elixir
# 在 config/runtime.exs 中配置
config :cortex, Cortex.Memory.TokenBudget,
  max_working_memory_tokens: 8000,
  observation_budget_ratio: 0.3,
  reflection_budget_ratio: 0.2
```

### 记忆触发阈值

```elixir
config :cortex, Cortex.Memory.Subconscious,
  reflection_trigger_threshold: 20,  # 消息数量
  observation_batch_size: 5,         # 批处理大小
  knowledge_graph_update_interval: 300  # 秒
```

### 手动控制

用户可以通过以下方式手动触发记忆操作：

- `记住这个解决方案` - 强制创建观察记录
- `总结一下我们的进展` - 触发反思处理
- `我偏好使用 X 方法` - 更新偏好设置
- `忘记关于 Y 的信息` - 清理特定记忆

## 存储位置

- **工作记忆**: 内存中，会话结束时清理
- **潜意识记忆**: SQLite 数据库 (`cortex_dev.db`)
- **知识图谱**: 向量索引 + 关系数据库
- **Tape 历史**: `./tape/<session_id>.jsonl`

## 隐私与安全

- 所有记忆数据本地存储，不上传云端
- 敏感信息（密码、密钥）自动过滤
- 支持选择性记忆清理和导出
- 遵循最小化数据收集原则
